# SwipeStats strategy artifacts

Interactive, standalone views for discussing SwipeStats' product and business
direction. Open any HTML file directly in a browser; each file contains its own
styles, interaction logic, and content.

## Views

- [`business-model.html`](business-model.html) — SwipeStats' connected consumer,
  media, and research-data model, with current evidence, hypotheses, tensions,
  and proposed validation.
- [`jobs-to-be-done.html`](jobs-to-be-done.html) — a user job map from immediate
  intent and export access through trustworthy interpretation, improvement, and
  longitudinal learning.
- [`evidence-to-iteration.html`](evidence-to-iteration.html) — candidate paths
  from customer evidence through desired outcomes and experiments to
  business-model consequences.

## How to use these

The views are discussion anchors, not authoritative product documentation.
Claims deliberately distinguish current product evidence from hypotheses and
open tensions. AI may update the pages as the product and evidence change, but
material changes should remain grounded in repository sources, real product
analytics, or user research.
